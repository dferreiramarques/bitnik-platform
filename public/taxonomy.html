Taxonomia de Nós

43 nós em 7 famílias — derivados empiricamente portando 4 jogos reais para a plataforma.
Cada nó é uma mecânica atómica. O grafo de nós é a definição completa do jogo.

Como funciona

Um jogo em Bitnik é um grafo de nós — nós ligados por edges. O runtime percorre o grafo seguindo as edges, chamando o executor de cada nó. Quando encontra um WAIT, para e aguarda input do jogador. Quando encontra END, o jogo termina.

Fluxo típico: GAME_START → ROUND_START → TURN_BLOCK → [player acts] → CHECK_WIN → (loop ou GAME_END)

ElementoO que éExemplo
NóMecânica atómica com ports de entrada e saídaTURN_BLOCK, DICE_ROLL
EdgeLigação sequencial entre ports — "depois disto, faz aquilo"TURN_END → next → CHECK_WIN
TriggerListener reactivo — "sempre que X, dispara Y"ON_DECK_E → fire → EXTRA_TURNS
SwimlaneRegião visual que organiza nós por fase (Setup / Loop / End)—

Edges vs Triggers

Edges modelam fluxo sequencial. Triggers modelam comportamento reactivo. A diferença importa: um trigger ON_DECK_E dispara mesmo que o jogo esteja no meio de um turno — a edge só seguiria quando o fluxo chegasse lá naturalmente.

Exemplo: endgame no Bulbous
ON_COND
reserve vazio → fire → CHECK_WIN → GAME_END

Quando um jogador coloca a última Baelfungious do reservatório, o trigger ON_COND dispara imediatamente — mesmo que o fluxo esteja dentro de TURN_BLOCK a aguardar apostas. Usar uma edge aqui seria errado: o CHECK_WIN só correria no fim do turno.

Turn Block — anatomia

O Turn Block é o nó mais importante. Substitui a necessidade de ligar Turn Start → [N acções] → Turn End. Define constraint, lista de acções, e outcomes que se tornam ports reais.

ConstraintComportamentoUsado em
choose_1Jogador escolhe exactamente 1 acçãoCapivaras (aposta), Praia das Percebes
up_to_nJogador faz até N acções (default: 2)Jogos de acção múltipla
allTodas obrigatórias em ordemNine Oils (rola → joga carta → resolve)
freeQualquer número, qualquer ordemSandbox, euro games
// Turn Block config no .bge
{
  "constraint": "up_to_n",
  "max": 2,
  "actions": [
    { "id": "a1", "type": "Play Card",  "mand": false },
    { "id": "a2", "type": "Move Piece", "mand": false }
  ],
  "outcomes": [
    { "id": "o1", "label": "end_turn", "color": "#4ade80" },
    { "id": "o2", "label": "endgame",  "color": "#f87171" }
  ]
}
// Cada outcome → port de saída real no nó

Swimlanes

As swimlanes dividem o canvas em fases. São cosméticas — não afectam a execução — mas impõem legibilidade e mapeiam directamente para scenes no runtime.

LaneProporção defaultO que contém
Setup20%GAME_START, DECK, distribuição inicial, roles iniciais
Game Loop60%ROUND_START, TURN_BLOCK, lógica de turno, triggers activos
End Game20%Scoring final, MAJORITY, COLL_BONUS, GAME_END

⚡ TRIGGERS Listeners reactivos

Triggers são sempre-on. Não precisam de estar ligados ao fluxo sequencial — o runtime indexa-os por tipo e dispara-os quando o evento ocorre, independentemente de onde o jogo está no grafo.

On Turn Start / End
Dispara no início/fim de qualquer turno de qualquer jogador.

On Round End
Dispara quando todos os jogadores completaram o turno numa ronda.
fire
TRIGGERS

Dispara quando tile é colocado. Config: filter por tipo (ex: type=salva-vidas).

TRIGGERS

Caso: Bulbous

Trick-taking com bluff. 2–4 jogadores. Padrão: Governor escolhe alvo → todos apostam → revelação → tie break.

Fluxo de um trick Bulbous
ROLE
Governante → TURN_BLOCK
escolhe alvo → SECRET_BET → SIMULTAN
revelar → CHECK_COND
empate? → TIMER 20s → SCORE_PTS
place bulb

Novos nós descobertos: SIMULTANEOUS (revelação paralela), ROLE (Governante rotativo), STRUCTURE (Baelfungious N-slot), TIMER (Tie Break 20s), MAJORITY (scoring de maioria nas Baelfungious completas).

Caso: Capivaras

Apostas secretas. 2–6 jogadores. Aposta única ganha — empate cancela.

Fluxo de uma ronda Capivaras
ROUND_START
vira N cartas → SECRET_BET
cada jogador aposta → SIMULTAN
revelar → COMP_CLAIM
único ganha → SCORE_PTS → CHECK_COND
baralho vazio?

Novos nós descobertos: SECRET_BET (aposta simultânea e secreta), COMPETING_CLAIM (único ganha, empate cancela, distinto de winner-takes-all), COLLECTION_BONUS (bónus por coleccionar N tipos).

Caso: Praia das Percebes

Tile placement. 2–4 jogadores. Tabuleiro 7×7 partilhado, salva-vidas marcam linhas/colunas.

Fluxo de um turno Praia das Percebes
TILE_PLACE
adjacente → ON_TILE
se salva-vidas → SPATIAL_SC
score segmento → FIRST_CLAIM
objectivo?

Novos nós descobertos: SHARED_BOARD (grid com adjacência), TILE_PLACE (validação ortogonal), SPATIAL_SCORE (scoring geométrico), FIRST_CLAIM (primeiro a cumprir reclama), PROGRESSIVE_REVEAL (pool de objectivos com janela de N visíveis), EXTRA_TURNS (última volta após endgame).

Caso: Nine Oils

Dados + combos + cartas modificadoras. 2 jogadores. Encher 6 garrafas antes do adversário bloquear tudo.

Fluxo de um turno Nine Oils
DICE_ROLL
9d6 → CARD_MOD
Temptress? → REACT_DEF
Preacher? → COMBO_DET → CONF_CHOICE
se ambíguo → SCORE_PTS
garrafas

Novos nós descobertos: DICE_ROLL (N dados, S faces), COMBO_DETECT (Double/Triple/Quad/Penta/Instant Win), CONFLICT_CHOICE (escolha entre bundles válidos), CARD_MODIFIER (altera próximo roll), REACTIVE_DEFENSE (resposta do oponente no teu turno), INSTANT_WIN (9-of-a-kind, bypassa Game End normal).

.bge — Formato

Ficheiro JSON portável. É a única fonte de verdade — o runtime lê directamente, o code generator converte para TypeScript.

{
  "_format": "bge",  // identificador fixo
  "_version": "0.4",
  "game": { "name": "...", "players": { "min": 2, "max": 4 } },

  "swimlanes": [
    { "id": "sl1", "name": "Setup",     "color": "#f5a623", "y": 0,    "h": 360 },
    { "id": "sl2", "name": "Game Loop", "color": "#2ec4a4", "y": 360,  "h": 1080 },
    { "id": "sl3", "name": "End Game",  "color": "#f87171", "y": 1440, "h": 360 }
  ],

  "nodes": [
    {
      "id": "n01",
      "node": "TURN_BLOCK",     // tipo do nó
      "label": "Turno do Jogador",
      "x": 280, "y": 430,          // posição no canvas
      "config": {
        "tb": {
          "constraint": "up_to_n", "max": 2,
          "actions":  [{ "id": "a1", "type": "Play Card", "mand": false }],
          "outcomes": [{ "id": "o1", "label": "end_turn", "color": "#4ade80" }]
        }
      }
    }
  ],

  "edges": [
    { "id": "e01", "fromNode": "n01", "fromPort": "end_turn",

}
